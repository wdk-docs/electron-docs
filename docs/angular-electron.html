angular-electron¶

使用Angular 8和Electron(Typescript + SASS + Hot Reload)引导和打包您的项目, 以创建桌面应用程序.

目前正在运行:

  • Angular v8.0.0

  • Electron v5.0.2

  • Electron Builder v20.41.0

有了这个样本, 你可以:

  • 使用Electron＆Hot重新加载在本地开发环境中运行您的应用程序

  • 在生产环境中运行您的应用程序

  • 将您的应用程序打包成Linux, Windows和Mac的可执行文件

Angular 8.0 CLI需要节点10.9或更高版本才能工作.

入门¶

在本地克隆此存储库:

git clone https://github.com/maximegris/angular-electron.git

使用npm安装依赖项:

npm install

yarn 和 node_modules 存在一个问题, 当应用程序由打包器构建时, 它只用在后端的 electron 中. 请使用 npm 作为依赖项管理器.

如果你想用Angular-cli生成Angular组件, 你 必须 在npm全局上下文中安装 @angular/cli. 如果您安装了以前版本的 angular-cli, 请按照 Angular-cli文档 进行操作.

npm install -g @angular/cli

为开发而构建¶

  • 在终端窗口 -> npm start

瞧！您可以在本地开发环境中使用Angular + Electron应用程序进行热重新加载！

应用程序代码由 main.ts 管理. 在此示例中, 应用程序使用简单的Angular App(http://localhost:4200)和Electron窗口运行. Angular组件包含Electron和NodeJS本机lib导入的示例. 您可以通过在 main.ts 中注释 win.webContents.openDevTools(); 来禁用 “开发者工具”.

包含的命令¶

命令

描述

npm run ng:serve:web

在浏览器中执行该应用程序

npm run build

构建应用程序. 您构建的文件位于 /dist 文件夹中.

npm run build:prod

使用Angular aot构建应用程序. 您构建的文件位于 /dist 文件夹中.

npm run electron:local

构建您的应用程序并启动 electron

npm run electron:linux

构建您的应用程序并在Linux系统上创建应用程序耗材

npm run electron:windows

在Windows操作系统上, 构建应用程序并在Windows 32/64位系统中创建应用程序耗材

npm run electron:mac

在MAC OS上, 构建应用程序并生成可在Mac上运行的应用程序的 .app 文件

您的应用程序已优化. 只有 /dist 文件夹和节点依赖项包含在可执行文件中.

你想在 electron 主线程中使用特定的lib(如rxjs)吗？¶

你可以这样做！只需使用 npm install --save 在npm依赖项(不是devDependencies)中导入库. 它将在构建阶段由 electron 加载并添加到最终封装中. 然后通过在 main.ts 文件中导入它来使用你的库. 容易没？

浏览器模式¶

也许你想在热重载的浏览器中执行应用程序？你可以用 npm run ng:serve:web 来做. 请注意, 在这种情况下, 您不能使用Electron或NodeJS本机库. 请检查 providers/electron.service.ts 以观察 electron/Native 库的条件导入是如何完成的.
